# INFITEX Global Advisory — Website

Static marketing website for INFITEX Global Advisory (white-label accounting &
bookkeeping outsourcing + Digitech for Australian practices and businesses).

Plain HTML + one CSS file + one JS file, generated by a small Python build system.
No framework, no runtime — the deployed output is fully static.

## Repo layout

```
/                     deployable site root (HTML + assets) — serve this folder
  *.html              generated pages (do NOT hand-edit; regenerate)
  styles.css          static stylesheet (edited directly)
  app.js              static JavaScript (edited directly)
  _headers            Cloudflare/Netlify cache + security headers
  _redirects          clean-URL redirects
  sitemap.xml robots.txt favicon.svg logo.svg og-image.png  infitex-contact-qr.*
  _build-sources/     the generator (build.py, _partials.py, validate*.py)
CONTEXT.md            what the site is, brand/compliance rules, architecture
INTEGRATION.md        deploy, forms, booking, SEO, rebuild + cache guide
```

## Deploy to Cloudflare Pages (Git)

1. Push this repo to GitHub.
2. Cloudflare dashboard → **Workers & Pages → Create → Pages → Connect to Git**.
3. Select the repo. **Build command:** *(leave empty)*. **Build output directory:** `/`.
4. Deploy, then add the custom domain under **Custom domains**.

The site is pre-built, so no build step runs on Cloudflare. (You can also use
**Upload assets** and drag in the repo contents.)

## Caching note (important)

`styles.css` and `app.js` are referenced with a content-hash version, e.g.
`styles.css?v=2b7db34c`. Each time their content changes, the build writes a new
hash, so browsers always fetch the latest after a deploy — no more "old theme on
first load" from a stale cached stylesheet. HTML is served `must-revalidate`, so
pages are always fresh. After the first deploy of this change, do one Cloudflare
**Caching → Purge Everything** to clear any old edge copies.

## Editing & rebuilding

- Page copy/markup → `_build-sources/build.py` (`build_<page>()`), then rebuild.
- Styling → `styles.css` (direct). Behaviour → `app.js` (direct).
- Shared header/footer/floaters/icons → `_build-sources/_partials.py`, then rebuild.

To rebuild (see INTEGRATION.md for detail): point `OUT` at the repo root in
`_partials.py`, `validate.py`, `validate2.py`; run `python3 build.py`; run both
validators (expect 0 issues / 0 warnings / 8 positive checks); restore `OUT`.
